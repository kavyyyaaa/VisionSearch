Retail Visual Indexing Case Study

Methodology & Setup

Problem Statement: Traditional keyword search fails when users search for specific patterns, collar layouts, silhouette profiles, or fabric details. Visual search maps these features directly into coordinates.

Feature Extraction: Images are processed using ResNet-50. The fully connected classification layer is stripped out, capturing the 2048-dimensional output of the final global average-pooling layer.

Similarity & Projection

FAISS Index: The extracted features are L2 normalized, enabling cosine similarity searches via dot products using FAISS `IndexFlatIP`. This returns exact ranked nearest neighbors extremely quickly.

Principal Component Analysis: To display the high-dimensional vectors, the server centered all cached product features and projected them onto the 2D plane dynamically using Singular Value Decomposition (SVD).

Limitations & Fixes

Model Biases: The generic ResNet model is pre-trained on general objects. It occasionally ranks items by background color or global silhouette profile rather than retail categories.

Mitigation: Visual search accuracy can be improved by fine-tuning the ResNet model using a metric learning approach (such as triplet loss) with a domain-specific fashion dataset.

Validation Protocol & Evaluation Methodology

Metrics are calculated using a **leave-one-out cross-validation** protocol over the 300 expanded catalog items. Test inputs are generated by applying random spatial crops, horizontal flips, and color/brightness distortions to database images, simulating real-world user photography. FAISS queries are dispatched, checking if the source catalog ID is correctly retrieved inside the Top-1 and Top-5 nearest neighbors.

Precision@K evaluates how many of the top K results share the correct target category, while **Recall@K** measures what percentage of all matching category variants are successfully returned within the top K neighbors.
